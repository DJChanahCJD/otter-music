1. 发现页的歌单市场 (Playlist Market)
实现机制：由前端直接处理网易云歌单请求，包含特定的参数加密（如 WEAPI/EAPI）与数据结构转换。

显示逻辑：

条件触发：仅在“搜索结果为空”或“清空搜索框重置”时显示。

极简原则：只保留核心分类，官方榜单（热歌、新歌等）采用硬编码 ID 维护，确保加载速度。

交互规范：

Tab 状态保持：采用 Keep-alive 或状态管理方案，确保在“发现”子页面切换 Tab 往返后，仍停留在具体的歌单详情页。

路由层级：点击歌单进入子页面，主导航 Tab 状态锁定在“发现”。

2. API 兼容性与拦截层 (Meting API Adapter)
API 拦截策略：拦截 source === '_netease' 的请求。

数据透传：

若 pic_id 或 url_id 已包含 http 协议前缀，前端直接返回原地址，避免二次处理导致的解析失效。

3. 跨源匹配转换系统 (Metadata Matching)
后台匹配逻辑：

针对 _netease 源歌曲，提取 Artist, Album, Title 信息。

重定向搜索：通过 GD Studio API（source: netease）进行二次检索。

信息覆盖：匹配成功后，用更高质量或更稳定的 url_id 等信息覆盖原始 _netease 数据。

触发场景：

自动提醒：若当前歌曲时长 ≤ 45s（判定为试听片段），弹出 Toast 提醒：“检测到试听片段，建议转换为 GD 音乐台播放”。

手动入口：播放器下拉菜单新增“转换为 GD 音乐台播放”及“查看歌曲来源”选项。

4. 扩展性预留 (Future Roadmap)
身份认证：预留网易云登录接口，后续通过 Cookie/Token 获取用户会员曲目权限。
