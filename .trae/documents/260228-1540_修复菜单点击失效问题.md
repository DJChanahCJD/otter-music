## 问题描述
前次修复（260228-1525）为了解决“点击菜单项误触列表行播放”的问题，在 `DrawerContent` 上使用了 `onClickCapture` 和 `onPointerDownCapture` 进行事件拦截。
然而，`Capture` 阶段的拦截过于激进，导致点击事件在到达菜单项按钮之前就被截断，使得菜单项（如“下一首播放”、“下载”等）无法响应点击。

## 修正方案
1. **移除 Capture 阶段拦截**：删除 `onClickCapture` 和 `onPointerDownCapture`。
2. **改为冒泡阶段拦截**：在 `DrawerContent` 上添加 `onClick={(e) => e.stopPropagation()}`。
   - 这样可以确保菜单项按钮先接收并处理点击事件（执行业务逻辑）。
   - 事件冒泡到 `DrawerContent` 时停止，不再继续向上传播到 `MusicTrackItem`（列表行），从而避免误触 `togglePlay`。
3. **移除 Pointer 事件拦截**：由于 `togglePlay` 仅绑定在 `onClick` 上，且 `Drawer` 组件的拖拽交互依赖 Pointer 事件，因此移除对 `onPointerDown` 的拦截，防止影响 Drawer 的拖拽关闭功能。

## 代码变更
文件：`src/components/MusicTrackMobileMenu.tsx`

```tsx
<DrawerContent
  // 移除: onPointerDownCapture={(e) => e.stopPropagation()}
  // 移除: onClickCapture={(e) => e.stopPropagation()}
  // 新增:
  onClick={(e) => e.stopPropagation()}
>
```

## 验证
1. **点击菜单项**：
   - 预期：菜单项逻辑（如 `onAddToNextPlay`）正常执行，且 Drawer 关闭。
   - 预期：列表行背景不会闪烁（未触发 `onPlay`），播放状态不改变。
2. **拖拽 Drawer**：
   - 预期：可以通过拖拽手柄或内容区域关闭 Drawer（Pointer 事件未被拦截）。
