> 提供匹配功能后，完全可以直接用网易云的搜索功能来精准搜索专辑、歌手。

这个思路非常**干练**。通过这种“**直接覆盖 + 状态标记**”的方式，你可以保持 `MusicTrack` 结构的高度一致性，减少了前端处理“替身对象”的复杂逻辑。

在这种方案下，一旦匹配成功，这首歌在数据库/本地存储中就变成了一首正常的、来自新源（如酷我、Joox）的歌曲，只是通过 `state` 字段保留了它的“出身证明”。

### 1. 更新后的数据结构

在 `src/types/music.ts` 中，我们只需最小化改动：

```typescript
export type TrackState = 'unmatched' | 'vip' | 'matching' | 'failed';

export interface MusicTrack {
  id: string;
  name: string;
  artist: string[];
  album: string;
  pic_id: string;
  url_id: string;
  lyric_id: string;
  source: MusicSource;  // _netease?
  // 新增可选字段
  state?: TrackState; 
}

```

### 2. 核心逻辑：即时转换 (JIT Conversion)

当用户点击播放时，我们执行以下流程：

1. **尝试播放**：调用 `musicApi.getUrl`。
2. **触发条件**：如果接口报错（404/下架）或返回的数据量极小（VIP 60s），则进入匹配流程。
3. **数据覆盖**：匹配成功后，直接用新音源的 `id`, `source`, `url_id` 等**覆盖**原有的字段，并**移除** `state` 字段。

### 3. 代码实现示例 (Play Controller)

你可以将这段逻辑集成在播放器的 `play` 方法中：

```typescript
async function playWithFallback(track: MusicTrack) {
  // 1. 如果是标记为待匹配的，或者正常的尝试播放
  try {
    const urlData = await musicApi.getUrl(track.source, track.url_id);

    // 检测是否为 VIP (假设 60s 限制)
    if (urlData.duration <= 60 && track.source === 'netease') {
      // 标记为 VIP，但不自动匹配（尊重你的需求：仅提醒，提供手动按钮）
      updateTrack(track.id, { state: 'vip' });
      toast("当前为 VIP 试听版");
      return startAudio(urlData.url);
    }

    // 正常播放
    return startAudio(urlData.url);

  } catch (error) {
    // 2. 无法播放且是导入歌曲，自动启用匹配
    if (track.source === 'netease' || track.state === 'unmatched') {
      return performAutoMatch(track);
    }
    toast.error("资源已失效");
  }
}

// 自动匹配并覆盖
async function performAutoMatch(track: MusicTrack) {
  updateTrack(track.id, { state: 'matching' });
  
  const query = `${track.name} ${track.artist[0]}`;
  const searchRes = await musicApi.searchAll(query, 1, 5);
  const bestMatch = findBestMatch(track, searchRes.items); // 使用之前的权重打分逻辑

  if (bestMatch) {
    // 【核心改进】：直接覆盖原数据，移除 state
    const { state, ...updatedData } = {
      ...track,
      ...bestMatch,
      id: track.id, // 保持本地 ID 不变，防止歌单关联失效
      // 可以保留原封面的 pic_id，如果觉得原封面更好看
    };
    
    saveToPlaylist(updatedData); // 持久化到本地存储
    toast.success(`已自动切换至 ${sourceLabels[bestMatch.source]} 音源`);
    return playWithFallback(updatedData);
  } else {
    updateTrack(track.id, { state: 'failed' });
    toast.error("未找到可用音源");
  }
}

```

### 4. UI 上的处理

在你的 `MusicTrackList` 组件中，只需要根据 `track.state` 渲染不同的微交互：

* **`state === 'vip'`**: 显示 👑 图标，点击图标或旁边的按钮手动触发 `performAutoMatch`。
* **`state === 'matching'`**: 显示 `Loader2` 旋转图标，禁用点击。
* **`state === 'failed'`**: 显示灰色感叹号，提醒用户该曲目全网难寻。
* **无 `state**`: 正常显示（即便它是从网易云转换过来的，它现在已经是一首合格的“酷我”或“Joox”歌曲了）。
